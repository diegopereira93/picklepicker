# Phase 12 Plan 02: Transactions and Retry Logic Summary

**One-liner:** Transaction-safe database connections with automatic rollback and resilient ML API calls using tenacity retry.

## What Was Built

### 1. Transaction Rollback Protection (Task 1)

Updated `pipeline/db/connection.py` to prevent pool poisoning from uncommitted transactions:

- `@asynccontextmanager` with try/except/rollback pattern
- Automatic `conn.rollback()` on any exception before re-raising
- Rollback failures logged at warning level (don't suppress original exception)
- Connection returned to pool in clean state

**Connection Pool Configuration:**
```python
max_waiting=10   # Queue up to 10 waiting connections (backpressure)
max_idle=300     # Close idle connections after 5 minutes
max_lifetime=3600 # Recycle connections after 1 hour
```

### 2. Mercado Livre Retry Logic (Task 2)

Added tenacity retry decorator to `search_pickleball_paddles()`:

- 3 retry attempts with exponential backoff (1s, 2s, 4s, max 10s)
- Retries on: `HTTPStatusError`, `ConnectError`, `TimeoutException`
- Does NOT retry on 4xx client errors (bad requests, auth failures)
- Security filter attached to logger with `scrub_sensitive_data`

### 3. Documentation Update (Task 3)

Added explanatory comment in `dropshot_brasil.py` clarifying automatic rollback behavior.
