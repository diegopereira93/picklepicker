# Phase 13 Plan 02: Button & Link Components

**Phase:** 13 — NVIDIA UI Redesign
**Plan:** 2 of 4
**Requirements:** NV-05, NV-08

## Goal

Replace the shadcn button variants with NVIDIA's exact button contracts (Primary, Secondary, Compact) and establish global link color rules for dark and light section contexts.

## Context

Depends on Plan 01 (CSS custom properties must exist). The current `button.tsx` uses `cva` + shadcn Tailwind variants. The component API (`variant`, `size`, `asChild`, `className`, `ref`) must be preserved exactly — call sites in `header.tsx`, `page.tsx`, and `paddles/page.tsx` pass `variant="outline"`, `variant="ghost"`, `size="sm"`, `size="lg"`, `size="icon"` — all must continue to render without prop errors. The strategy is to keep the `cva` infrastructure but rewrite the variant CSS values to NVIDIA contracts.

Link styles are added as CSS rules in `globals.css` scoped to `.nv-dark-section a` and `.nv-light-section a` — no changes to individual anchor elements across the codebase are required.

## Tasks

### Task 1: Rewrite button.tsx with NVIDIA button contracts

**File:** `frontend/src/components/ui/button.tsx`
**Action:** edit

Preserve all imports (`React`, `Slot`, `cva`, `VariantProps`, `cn`). Preserve the `ButtonProps` interface and `Button` forwardRef component unchanged. Only rewrite the `buttonVariants` cva definition.

The base class string must be replaced with NVIDIA base styles:

```
"inline-flex items-center justify-center whitespace-nowrap font-bold
 rounded-[2px] transition-colors
 disabled:pointer-events-none disabled:opacity-50
 focus-visible:outline focus-visible:outline-2 focus-visible:outline-[#000000]
 focus-visible:outline-offset-2"
```

The `variants` object must implement:

**variant:**
- `default` (maps to NVIDIA Primary): `"bg-transparent border-[2px] border-[#76b900] text-white hover:bg-[#1eaedb] hover:text-white active:bg-[#007fff] active:border-[#003eff] active:border rounded-[2px] focus-visible:bg-[#1eaedb] focus-visible:opacity-90"`
- `secondary` (maps to NVIDIA Secondary): `"bg-transparent border border-[#76b900] text-white hover:bg-[#1eaedb] hover:text-white active:bg-[#007fff] active:border-[#003eff] rounded-[2px] focus-visible:bg-[#1eaedb] focus-visible:opacity-90"`
- `compact` (maps to NVIDIA Compact): `"bg-transparent border border-[#76b900] text-white hover:bg-[#1eaedb] hover:text-white rounded-[2px] tracking-[0.144px] leading-none"`
- `outline`: alias to secondary — `"bg-transparent border border-[#76b900] text-white hover:bg-[#1eaedb] hover:text-white rounded-[2px]"`
- `ghost`: `"bg-transparent text-white hover:bg-[#1a1a1a] hover:text-white rounded-[2px]"`
- `destructive`: `"bg-[#e52020] text-white border-0 hover:bg-[#c01818] rounded-[2px]"`
- `link`: `"bg-transparent text-[#76b900] underline hover:text-[#3860be] p-0 h-auto"`

**size:**
- `default`: `"py-[11px] px-[13px] text-[16px] font-bold leading-[1.25]"`
- `sm`: `"py-[11px] px-[13px] text-[14px] font-bold leading-[1.25]"`
- `lg`: `"py-[11px] px-[13px] text-[16px] font-bold leading-[1.25]"`
- `icon`: `"h-10 w-10 p-0"`
- `compact`: `"py-[11px] px-[13px] text-[16px] font-bold leading-none tracking-[0.144px]"`

defaultVariants remain `{ variant: "default", size: "default" }`.

Keep `Button.displayName = "Button"` and `export { Button, buttonVariants }` unchanged.

**Verification:** `grep -c "76b900" frontend/src/components/ui/button.tsx` returns at least 3. `grep "11px" frontend/src/components/ui/button.tsx` returns at least 1 match.

### Task 2: Add global link styles to globals.css

**File:** `frontend/src/app/globals.css`
**Action:** edit — append new rules at the end of the file (after existing content)

Add the following CSS block. It uses the `.nv-dark-section` and `.nv-light-section` helper classes defined in Plan 01 to scope link behavior:

```css
/* === Global link styles (NV-08) === */

/* Links on dark backgrounds (#000000 / #1a1a1a sections) */
.nv-dark-section a,
a.nv-link-dark {
  color: #ffffff;
  text-decoration: none;
}

.nv-dark-section a:hover,
a.nv-link-dark:hover {
  color: #3860be;
  text-decoration: none;
}

/* Links on light backgrounds (#ffffff sections) */
.nv-light-section a,
a.nv-link-light {
  color: #000000;
  text-decoration: underline;
  text-decoration-color: #76b900;
  text-decoration-thickness: 2px;
  text-underline-offset: 2px;
}

.nv-light-section a:hover,
a.nv-link-light:hover {
  color: #3860be;
  text-decoration: none;
}

/* Green links (standalone accent links) */
a.nv-link-green {
  color: #76b900;
  text-decoration: none;
}

a.nv-link-green:hover {
  color: #3860be;
  text-decoration: none;
}

/* Muted links */
a.nv-link-muted {
  color: #666666;
  text-decoration: none;
}

a.nv-link-muted:hover {
  color: #3860be;
  text-decoration: none;
}
```

**Verification:** `grep -c "3860be" frontend/src/app/globals.css` returns at least 4 (one for each link hover rule).
