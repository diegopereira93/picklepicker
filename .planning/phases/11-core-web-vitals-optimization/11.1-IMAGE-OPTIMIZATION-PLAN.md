# Plan 11.1: Image Optimization Audit & Fixes

## Objective
Audit current image usage and migrate from raw `<img>` tags to Next.js `next/image` for LCP and CLS improvement. Achieve WebP/AVIF automatic format serving, responsive sizes, and priority loading for hero images.

## Scope

### In Scope
- Audit all `<img>` tags across the app (not using next/image)
- Migrate to `next/image` with proper width/height attributes
- Add priority loading for above-the-fold images
- Configure responsive image sizes
- Add preload hints for hero images

### Out of Scope
- Image CDN migration (Next.js built-in sufficient)
- Manual WebP/AVIF conversion (automatic via next/image)
- Changing image aspect ratios or styling

## Success Criteria
1. All raw `<img>` tags migrated to `next/image`
2. All images have explicit width and height attributes
3. Hero/above-the-fold images have `priority` prop
4. Responsive images use `sizes` prop appropriately
5. Lighthouse LCP score improves to < 2.5s target

## Technical Context

### Current State
- Frontend uses raw `<img>` tags in `/paddles/page.tsx` and `/paddles/[brand]/[model-slug]/page.tsx`
- No width/height attributes causing layout shifts
- No priority loading for hero images
- No image preloading

### Target State
- All images use Next.js `Image` component with dimensions
- Hero images load with priority
- Responsive sizes configured for mobile/tablet/desktop
- Hero images preloaded in layout

</read_first>

<action>
1. Update paddle-card.tsx to use Image component:

     alt={paddle.name}
     width={480}
     height={480}

### Task 3: Add Priority Loading and Preload for Hero Images

**Objective:** Configure priority loading for above-the-fold images and preload hints

**Prerequisites:** Task 2 complete (Image component migration)

**Files:**

<read_first>
- frontend/src/app/layout.tsx
- frontend/src/app/paddles/[brand]/[model-slug]/page.tsx
</read_first>

<action>
1. Add priority prop to hero images:
   ```tsx
   // In paddle detail page, first/main image:
   <Image
     src={paddle.image_url}
     alt={`${paddle.brand} ${paddle.model}`}
     width={600}
     height={600}
     priority={true}
     sizes="(max-width: 768px) 100vw, 50vw"
   />
   ```

2. Add preload hint for critical hero image in layout.tsx:
   ```tsx
   // In metadata or as Link component:
   export const metadata = {
     // ... existing metadata
     other: {
       preload: [
         { rel: 'preload', as: 'image', href: '/hero-default.webp' }
       ]
     }
   };
   ```

3. Configure sizes prop strategically:
   - Mobile (< 768px): 100vw for hero, 50vw for grid
   - Tablet (768-1024px): 50vw for hero, 33vw for grid
   - Desktop (> 1024px): 50vw for hero, 25vw for grid
</action>
