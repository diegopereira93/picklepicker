## Decisions Made

1. **Speed Insights sample rate:** Set to 1.0 (100%) for small site; can reduce to 0.1 for high traffic
2. **Bundle analyzer:** Dynamic import pattern to avoid loading in production
3. **Performance budget:** 150KB total JS based on Next.js best practices for e-commerce sites
4. **Focus indicators:** Using primary HSL color with outline-offset for visibility
5. **Announcer hook:** Polite (non-interrupting) aria-live region for dynamic content
6. **Semantic HTML:** Converted ProductCard from div to article for better screen reader context

## How to Use

### Run Bundle Analyzer
```bash
cd frontend
npm run analyze
```

### Check Bundle Size
```bash
cd frontend
npm run size
```

### Run Lighthouse CI Locally
```bash
cd frontend
npm run lighthouse:ci
```

### Use Screen Reader Announcer
```tsx
import { useAnnouncer, Announcer } from '@/hooks/use-announcer';

function MyComponent() {
  const { announce, announcement } = useAnnouncer();

  useEffect(() => {
    announce('Content loaded successfully');
  }, []);

  return (
    <>
      <Announcer announcement={announcement} />
      {/* component content */}
    </>
  );
}
```
