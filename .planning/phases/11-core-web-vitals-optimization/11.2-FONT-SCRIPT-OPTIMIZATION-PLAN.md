# Plan 11.2: Font & Script Optimization

## Objective
Optimize font loading strategy with display swap and fallback adjustments. Audit and configure third-party script loading strategies to reduce TBT/INP impact.

## Scope

### In Scope
- Audit current font configuration in layout.tsx
- Add display: 'swap' and adjustFontFallback to next/font
- Audit all third-party scripts loading on site
- Configure Script component with lazyOnload strategy
- Defer non-critical analytics and widgets

### Out of Scope
- PartyTown integration (deferred to future phase if needed)
- Removing third-party scripts (only changing load strategy)
- Custom font hosting migration

## Success Criteria
1. Fonts use display: 'swap' with adjustFontFallback
2. Critical fonts preloaded in layout
3. All third-party scripts use appropriate loading strategies
4. Non-critical scripts deferred with lazyOnload
5. No render-blocking scripts remain
6. Lighthouse TBT score improves

## Technical Context

### Current State
- Font configuration exists but missing display: 'swap'
- No font preloading configured
- Third-party scripts (Clerk, etc.) loading without strategy
- No script loading audit performed

### Target State
- next/font configured with display: 'swap' and adjustFontFallback: true
- Critical fonts preloaded
- Third-party scripts use next/script with proper strategies
- Analytics deferred until after hydration

## Tasks

### Task 1: Optimize Font Loading Configuration

**Objective:** Configure fonts with swap display and fallback adjustment

**Prerequisites:** None

**Files:**

<read_first>
- frontend/src/app/layout.tsx
- frontend/tailwind.config.ts
</read_first>

<action>
1. Update font configuration in layout.tsx:
   ```tsx
   import { Inter } from 'next/font/google';

   const inter = Inter({
     subsets: ['latin'],
     display: 'swap',              // Prevents invisible text during load
     variable: '--font-sans',
     preload: true,
     adjustFontFallback: true,     // Reduces layout shift from font swap
     fallback: ['system-ui', 'sans-serif']
   });
   ```

2. Add font preload to metadata:
   ```tsx
   export const metadata = {
     // ... existing metadata
     other: {
       preconnect: [
         { rel: 'preconnect', href: 'https://fonts.googleapis.com' },
         { rel: 'preconnect', href: 'https://fonts.gstatic.com', crossOrigin: 'anonymous' }
       ]
     }
   };
   ```

3. Verify font variable applied to body:
   ```tsx
   <body className={`${inter.variable} font-sans antialiased`}>
   ```
</action>

<verify>
- [ ] Font config includes display: 'swap'
- [ ] Font config includes adjustFontFallback: true
- [ ] Preconnect hints added for Google Fonts
- [ ] Font variable applied to body element
</verify>

### Task 3: Configure Script Loading Strategies

**Objective:** Apply next/script with proper loading strategies

**Prerequisites:** Task 2 complete (script audit)

**Files:**

<read_first>
- frontend/src/app/layout.tsx
- frontend/src/app/providers.tsx
</read_first>

<action>
1. Import Script component in layout:
   ```tsx
   import Script from 'next/script';
   ```

2. Add non-critical scripts with lazyOnload:
   ```tsx
   {/* Analytics - load after page becomes idle */}
   <Script
     src="https://analytics.example.com/script.js"
     strategy="lazyOnload"
     id="analytics-script"
   />

   {/* Optional: Third-party widgets after interactive */}
   <Script
     src="https://widget.example.com/widget.js"
     strategy="afterInteractive"
     id="widget-script"
   />
   ```

3. Configure Clerk provider with strategy considerations:
   ```tsx
   {/* Clerk is critical for auth - keep eager but verify */}
   <ClerkProvider>
     {children}
   </ClerkProvider>
   ```

4. Add beforeInteractive for critical analytics if needed:
   ```tsx
   {/* Only for truly critical scripts */}
   <Script
     src="https://critical-cdn.com/script.js"
     strategy="beforeInteractive"
   />
   ```
</action>

## Notes

### Key Implementation Details
- display: 'swap' prevents invisible text during font load (FONT-01)
- adjustFontFallback reduces layout shift from font swap (CLS-03)
- lazyOnload defers scripts until after page idle (SCRIPT-02)
- Latin subset minimizes font file size (FONT-03)

### Script Strategy Reference
- beforeInteractive: Critical, render-blocking (use sparingly)
- afterInteractive: Load after hydration (widgets, non-critical)
- lazyOnload: Load when browser idle (analytics, tracking)
