**Prerequisites:** None

**Files:**

<read_first>

- frontend/package.json
</read_first>

<action>
1. Install Speed Insights package:
   ```bash
   cd frontend && npm install @vercel/speed-insights
   ```

2. Add Speed Insights component to layout:
   ```tsx
   // app/layout.tsx
   import { SpeedInsights } from '@vercel/speed-insights/next';

   export default function RootLayout({ children }) {
     return (
       <html lang="pt-BR">
         <body>
           {children}
           <SpeedInsights sampleRate={1.0} />
         </body>
       </html>
     );
   }
   ```

   ```tsx
   <SpeedInsights

   />
   ```

4. Verify build succeeds:
   ```bash
   npm run build
   ```
</action>

### Task 2: Set Up Bundle Analyzer and Performance Budget

**Objective:** Configure bundle size tracking and CI performance budget

**Prerequisites:** None

**Files:**

<read_first>
- frontend/next.config.mjs
- frontend/package.json
</read_first>

<action>
1. Install bundle analyzer:
   ```bash
   cd frontend && npm install -D @next/bundle-analyzer
   ```

2. Update next.config.mjs:
   ```javascript
   import withBundleAnalyzer from '@next/bundle-analyzer';

   const nextConfig = {
     // ... existing config
   };

   const bundleAnalyzerConfig = withBundleAnalyzer({
     enabled: process.env.ANALYZE === 'true',
   });

   export default process.env.ANALYZE === 'true'
     ? bundleAnalyzerConfig(nextConfig)
     : nextConfig;
   ```

3. Add analyze script to package.json:
   ```json
   {
     "scripts": {
       "analyze": "cross-env ANALYZE=true next build"
     }
   }
   ```

4. Install size-limit for performance budget:
   ```bash
   npm install -D size-limit @size-limit/preset-app
   ```

5. Add size-limit config to package.json:
   ```json
   {
     "size-limit": [
       {
         "path": ".next/static/chunks/**/*.js",
         "limit": "150 KB"
       },
       {
         "path": ".next/static/chunks/pages/_app-*.js",
         "limit": "50 KB"
       }
     ]
   }
   ```

6. Add size-limit script:
   ```json
   {
     "scripts": {
       "size": "size-limit"
     }
   }
   ```
</action>

<verify>
- [ ] @next/bundle-analyzer in devDependencies
- [ ] ANALYZE env var controls bundle analyzer
- [ ] size-limit configured with 150KB limit
- [ ] npm run analyze opens bundle analyzer
</verify>

### Task 3: Configure Lighthouse CI

**Objective:** Add Lighthouse CI for automated performance testing

**Prerequisites:** Task 2 complete (build configuration)

**Files:**

<read_first>
- frontend/package.json
- .github/workflows/ (check existing workflows)
</read_first>

<action>
1. Install Lighthouse CI:
   ```bash
   cd frontend && npm install -D @lhci/cli
   ```

2. Create lighthouserc.js config:
   ```javascript
   // frontend/lighthouserc.js
   module.exports = {
     ci: {
       collect: {
         url: [
           'http://localhost:3000/',
           'http://localhost:3000/paddles'
         ],
         numberOfRuns: 3,
       },
       assert: {
         assertions: {
           'categories:performance': ['warn', { minScore: 0.9 }],
           'categories:accessibility': ['error', { minScore: 0.9 }],
           'largest-contentful-paint': ['error', { maxNumericValue: 2500 }],
           'cumulative-layout-shift': ['error', { maxNumericValue: 0.1 }],
           'total-blocking-time': ['warn', { maxNumericValue: 200 }],
         },
       },
     },
   };
   ```

3. Add Lighthouse CI script:
   ```json
   {
     "scripts": {
       "lighthouse": "lhci autorun"
     }
   }
   ```

4. Create GitHub Actions workflow:
   ```yaml
   # .github/workflows/lighthouse.yml
   name: Lighthouse CI
   on: [push]
   jobs:
     lighthouse:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: 20
         - run: npm ci
         - run: npm run build
         - run: npm run lighthouse
   ```
</action>

<verify>
- [ ] lighthouserc.js exists with assertions
- [ ] Performance budget: LCP < 2.5s

</verify>

### Task 5: Fix Accessibility Violations

**Objective:** Fix all WCAG 2.1 AA violations found in audit

**Prerequisites:** Task 4 complete (audit results)

**Files:**

<read_first>
- Files identified in audit with violations
</read_first>

<action>
1. Fix alt text for all images:
   ```tsx
   // Before: <Image src={url} alt="" />
   // After: <Image src={url} alt={`${brand} ${model} pickleball paddle`} />
   ```

2. Add visible focus indicators:
   ```css
   /* In globals.css or Tailwind config */
   *:focus-visible {
     outline: 2px solid hsl(var(--primary));
     outline-offset: 2px;
   }
   ```

3. Fix color contrast issues:
   ```tsx
   // Before: text-gray-400 on white (fails contrast)
   // After: text-gray-600 (passes 4.5:1)
   ```

4. Ensure keyboard navigation:
   ```tsx
   // Add tabindex and keyboard handlers where needed
   <div
     role="button"
     tabIndex={0}
     onKeyDown={(e) => e.key === 'Enter' && handleClick()}
   >
   ```

5. Add screen reader announcements for dynamic content:
   ```tsx
   import { useAnnouncer } from '@/hooks/use-announcer';

   // Announce when content updates
   useAnnouncer('Products loaded successfully');
   ```
</action>
