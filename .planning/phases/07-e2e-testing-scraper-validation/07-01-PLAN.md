# v1.1 Phase 1: E2E Testing & Scraper Validation

**Objective:** Validate all 3 scrapers (Brazil Store, Drop Shot Brasil, Mercado Livre) + Firecrawl integration work correctly locally with staging/test data, achieving ≥80% code coverage and documenting error handling behavior.

**Phase Goal:** De-risk production scraper deployment by comprehensive local E2E validation before scaling to production cron schedule.

**Success Criteria (ALL must be TRUE):**
1. ✓ E2E test suite covers all 3 scrapers with ≥80% code path coverage (pytest --cov output)
2. ✓ Each scraper tested against real retailer URL (or staging mock) with schema validation
3. ✓ Firecrawl `/extract` error modes documented and handled (timeout, rate limit, parse failure)
4. ✓ Data integrity verified: schema compliance, dedup matching, affiliate URL formatting
5. ✓ Performance validated: crawl times <30s per retailer, no memory leaks
