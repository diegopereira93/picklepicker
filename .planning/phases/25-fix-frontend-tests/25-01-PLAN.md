The test's localStorage mock (lines 107-113) does NOT set this key, so `getAuthToken()` returns null, and `migrateProfileOnLogin` throws `"Not authenticated"` at line 96 — before it even reaches the fetch call.

  **Fix:** In the `beforeEach` block (line 104), add `__clerk_client_jwt` to the localStorage mock so `getAuthToken()` returns a truthy value. Add this after the localStorage mock definition:

  ```typescript
  beforeEach(() => {
    vi.clearAllMocks()
    const localStorageMock = {
      getItem: vi.fn((key: string) => {
        if (key === '__clerk_client_jwt') return 'mock-jwt-token'
        return null
      }),
      setItem: vi.fn(),
      removeItem: vi.fn(),
      clear: vi.fn(),
    }
    vi.stubGlobal('localStorage', localStorageMock)
  })
  ```

  This makes `getAuthToken()` return `'mock-jwt-token'`, which allows `migrateProfileOnLogin` to proceed past the auth check and reach the `fetch` call (which is already mocked at line 14-15).
