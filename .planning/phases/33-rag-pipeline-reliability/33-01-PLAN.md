<objective>
Fix the two most dangerous silent failures in the RAG pipeline: zero-vector embedding fallback and LLM timeout that's too short.

Purpose: When embedding providers fail, users currently get random results disguised as recommendations. When Groq times out at 8s, complex queries silently degrade. This plan makes both failures explicit and handleable.

Output: EmbeddingUnavailableError raised instead of zero vector, 15s LLM timeout with streaming awareness, timeout alerts via Telegram.
</objective>

class EmbeddingUnavailableError(Exception):
