  <files>backend/Dockerfile</files>
  <action>
Create `backend/Dockerfile` with multi-stage build for production. The Dockerfile MUST:

1. Use `python:3.12-slim` as base image (slim = smaller, no unnecessary build tools)
2. Set working directory to `/app`
3. Copy `backend/pyproject.toml` first for dependency caching (Docker layer cache optimization)
4. Install dependencies with `pip install .` (uses pyproject.toml)
5. Copy the rest of `backend/` source code
6. Expose the PORT env var (Railway sets this dynamically)
7. Use the SAME start command as railway.toml: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
8. Set `PYTHONUNBUFFERED=1` for proper log flushing
9. Run as non-root user for security

IMPORTANT: The Dockerfile context is the repo root (Railway builds from root), but the `dockerfile` path is `backend/Dockerfile`. So COPY paths are relative to repo root.

```dockerfile
FROM python:3.12-slim AS base

# Prevent Python from writing .pyc files and enable unbuffered output
ENV PYTHONDONTWRITEBYTECODE=1
ENV PYTHONUNBUFFERED=1

WORKDIR /app

# Install system dependencies for psycopg (binary wheels need libpq)
RUN apt-get update && apt-get install -y --no-install-recommends \
    libpq-dev \
    && rm -rf /var/lib/apt/lists/*

# Copy dependency file first (layer cache)
COPY backend/pyproject.toml ./backend/pyproject.toml

# Install Python dependencies
RUN pip install --no-cache-dir ./backend

# Copy backend source
COPY backend/ ./backend/

# Create non-root user
RUN adduser --disabled-password --gecos "" appuser
USER appuser

# Railway sets PORT dynamically
EXPOSE ${PORT:-8000}

# Start FastAPI (matches railway.toml [start] cmd)
CMD ["sh", "-c", "uvicorn backend.app.main:app --host 0.0.0.0 --port ${PORT:-8000}"]
```

Note on CMD: Use `sh -c` to expand the $PORT variable. The railway.toml `cmd` also uses `$PORT` expansion. The module path is `backend.app.main:app` since WORKDIR is `/app` and backend/ is a subdirectory. However, railway.toml uses `app.main:app` — verify which is correct for the deployment context. If Railway sets WORKDIR differently, the CMD may need adjustment. Include both patterns as a comment.
