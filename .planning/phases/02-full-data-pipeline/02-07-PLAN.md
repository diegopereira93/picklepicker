1. Create `backend/app/db.py` — a backend-specific DB connection module following the same pattern as `pipeline/db/connection.py` (psycopg AsyncConnectionPool singleton). This is a separate module because backend runs as a FastAPI service (managed lifecycle) while pipeline runs as one-shot scripts.

```python
import os
from contextlib import asynccontextmanager
from psycopg_pool import AsyncConnectionPool

_pool: AsyncConnectionPool | None = None

async def get_pool() -> AsyncConnectionPool:
    global _pool
    if _pool is None:
        _pool = AsyncConnectionPool(
            conninfo=os.environ["DATABASE_URL"],
            min_size=2,
            max_size=10,
        )
        await _pool.open()
    return _pool

@asynccontextmanager
async def get_connection():
    pool = await get_pool()
    async with pool.connection() as conn:
        yield conn

async def close_pool():
    global _pool
    if _pool is not None:
        await _pool.close()
        _pool = None
```

Note: Backend uses min_size=2, max_size=10 (higher than pipeline's 1/5) because it serves concurrent HTTP requests.

Critical implementation details:
- psycopg3 uses `%s` placeholders (not `$1`, `$2`). Replace ALL `$N` placeholders with `%s` in the SQL queries. The current queries use `${len(params)+1}` pattern which is PostgreSQL native — psycopg3 translates `%s` to `$N` internally.
- Use `conn.execute(query, params)` where params is a tuple/list — psycopg3 handles parameterization.
- Use `cursor.fetchall()` for list results and `cursor.fetchone()` for single results.
- Set `row_factory=dict_row` on the cursor so rows are dicts (matching the existing `.get()` access pattern in endpoint code).

Example pattern for list_paddles:
```python
from psycopg.rows import dict_row

async with get_connection() as conn:

        count_row = await cur.fetchone()
        total = count_row["total"] if count_row else 0

        await cur.execute(data_query, params + [limit, offset])
```
