_jwks_cache: Optional[dict] = None
_jwks_cache_time: float = 0.0

4. **`get_cached_jwks()` async function**: 
   - If `_jwks_cache` is not None AND `time.time() - _jwks_cache_time < JWKS_CACHE_TTL`, return cached `_jwks_cache`.
   - Else: `response = requests.get(CLERK_JWKS_URL, timeout=10)`, `response.raise_for_status()`, set `_jwks_cache = response.json()`, `_jwks_cache_time = time.time()`, return `_jwks_cache`.

5. **`ClerkAuthState` dataclass**:
```python
@dataclass
class ClerkAuthState:
    clerk_id: str
    email: Optional[str] = None
    payload: Optional[dict] = None
```

6. **`verify_clerk_token(token: str)` async function**:
   - Get unverified header: `unverified_header = jwt.get_unverified_header(token)`
   - Extract `kid` from header
   - Call `jwks = await get_cached_jwks()`
   - Find matching key: iterate `jwks.get("keys", [])`, match `key.get("kid") == kid`
   - Get public key: `public_key = jwt_algorithms.RSAAlgorithm.from_jwk(key)` (where `key` is the matched dict)
   - If no key found: `raise HTTPException(status_code=401, detail="Token inválido", headers={"X-Error-Code": "INVALID_KEY"})`
   - Decode: `payload = jwt.decode(token, public_key, algorithms=["RS256"], issuer=CLERK_ISSUER, options={"verify_exp": True, "verify_iss": True, "verify_signature": True})`
   - On `jwt.ExpiredSignatureError`: `raise HTTPException(status_code=401, detail="Token expirado")`
   - On `jwt.DecodeError`: `raise HTTPException(status_code=401, detail="Token mal formatado")`
   - On `Exception as e`: `raise HTTPException(status_code=401, detail=f"Token inválido: {str(e)}")`
   - Extract `clerk_id = payload.get("sub")`
   - If no `clerk_id`: `raise HTTPException(status_code=401, detail="Token sem identificador de usuário")`
   - Return `ClerkAuthState(clerk_id=clerk_id, email=payload.get("email"), payload=payload)`

7. **`require_clerk_auth(request: Request)` async function** (the FastAPI Depends dependency):
   - `auth_header = request.headers.get("Authorization")`
   - If not auth_header: `raise HTTPException(status_code=401, detail="Header Authorization requerido")`
   - If not `auth_header.lower().startswith("bearer ")`: `raise HTTPException(status_code=401, detail="Formato Authorization deve ser 'Bearer <token>')`
   - `token = auth_header[7:]` (strip "Bearer ")
   - If not token: `raise HTTPException(status_code=401, detail="Token vazio")`
   - If `CLERK_JWKS_URL` is empty: `raise HTTPException(status_code=503, detail="Serviço de autenticação não configurado")`
   - Return `await verify_clerk_token(token)`

8. **Optional `get_optional_clerk_auth(request: Request)` async function**:
   - If no Authorization header: return `None`
   - If "Bearer ": return `None` (empty token → optional, not required)
   - If valid token: return `await verify_clerk_token(token)` (same flow as require_clerk_auth)
   - This is useful if any endpoint wants to accept optional auth.

All error messages in PT-BR. All HTTPException responses include `headers={"X-Error-Code": "..."}` for client-side error handling.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-03 | Spoofing | JWT forgery | mitigate | RS256 signature verification via JWKS (PyJWT + cryptography) — only Clerk can sign valid tokens |
| T-32-04 | Tampering | JWT replay | mitigate | JWT `exp` claim verified by PyJWT — expired tokens rejected |
| T-32-05 | Repudiation | Unauthenticated access | mitigate | Protected endpoints return 401 without valid Clerk JWT; all auth failures logged with X-Error-Code headers |
| T-32-06 | Information Disclosure | JWKS per-request fetch | mitigate | JWKS cached for 1 hour (_jwks_cache with TTL) — no per-request external calls |
| T-32-07 | Denial | Clerk JWKS unavailable | accept | If Clerk JWKS endpoint fails, auth returns 503 "não configurado"; public endpoints unaffected |
| T-32-08 | Elevation | User accesses other user's data | mitigate | Users can only access their own data (clerk_id from verified token, not from request params) |
</threat_model>
