3. After the existing model_slug query, if `paddle` is null AND `modelSlug` matches `/^\d+$/`, perform a fallback fetch by numeric ID:

The full replacement for `fetchProductData` (lines 7-14) should be:

```typescript
export async function fetchProductData(brand: string, modelSlug: string): Promise<Paddle | null> {
  const res = await fetch(
    `${FASTAPI_URL}/api/v1/paddles?brand=${encodeURIComponent(brand)}&model_slug=${encodeURIComponent(modelSlug)}`
  )
  if (!res.ok) return null
  const data = await res.json()
  const paddle = data.data?.[0] ?? data.paddles?.[0] ?? null

  if (!paddle && /^\d+$/.test(modelSlug)) {
    const idRes = await fetch(`${FASTAPI_URL}/api/v1/paddles/${modelSlug}`)
    if (idRes.ok) {
      const idData = await idRes.json()
      return idData.data ?? idData ?? null
    }
  }

  return paddle
}
```

Key changes from original:
- `throw new Error(...)` on non-ok response becomes `return null` (graceful degradation)
- After the slug-based query returns null, a numeric ID fallback fetch is attempted
- The fallback uses the direct `/api/v1/paddles/{id}` endpoint which already exists in FastAPI

Per CONTEXT.md decision 5. Do NOT modify `fetchPaddlesList` or `generateProductMetadata`.
