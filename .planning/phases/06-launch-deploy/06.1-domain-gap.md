# Gap Closure 06.1: Acquire & Configure Production Domain

## Objective

Register production domain (pickleiq.com) and configure DNS/SSL with Vercel and Railway. Currently using Vercel's auto-generated domain (picklepicker.vercel.app) and need to switch to production.

## Problem

Phase 06 verification assumed pickleiq.com was registered and configured, but domain was never purchased. Current state:
- Frontend: `picklepicker.vercel.app` (Vercel auto-domain)
- API: Railway auto-generated URL
- VERIFICATION.md URLs are incorrect/aspirational

## Solution

**Option A (Recommended): Register pickleiq.com**
- Cost: ~$15-20/year
- Steps:
  1. Namecheap / Vercel Domains / GoDaddy → register pickleiq.com
  2. Vercel Dashboard → Settings → Domains → Add pickleiq.com
  3. Update .env.example with real URLs
  4. Re-run Playwright validation against production domain
  5. Update vercel.json with final domain

**Option B (Temporary): Use staging domains**
- Cost: $0
- Steps:
  1. Get actual Vercel deployment URL (picklepicker.vercel.app)
  2. Get actual Railway URL (e.g., api-xyz.railway.app)
  3. Update .env.example with staging URLs
  4. Update vercel.json rewrites to point to real Railway URL
  5. Test against staging domain
  6. Plan future migration to production domain (Phase 07)
